Semantics

HTML5 introduced plenty of new tags that give meaning to our website's content and solves the issue of 'div-mania'.

Example:

<nav> This is our nav </nav>
<main> Our main content is here <main>
<footer> Nice footer tag </footer>

Common semantic tags:

  • Main tag -> main content
  • Nav tag -> for the website's navigation
  • Header tag -> head content
  • Footer tag -> footer content
  • Aside tag -> mostly used for sidebars
  • Section tag -> to separate our content
  • Article tag -> for each article
  • Audio tag -> contains audio
  • Video tag -> contains video
  • Output tag -> result of a calculation

And a lot more!

Structure

The main principle of a semantic structure is that it should be logical: a DOCTYPE of html must be declared, the html tag should have a language attribute, semantic tags must be used, headings should follow a hierarchial order, etc.

Example:

<!DOCTYPE html>
<html lang="en"></html>

Landmarks

The ARIA (Accessible Rich Internet Application) landmarks provide specific attributes to help assistive technologies to navigate a website, it does so by assinging roles to the HTML5 semantic tags. The best practice is to use both: semantic tags and landmark roles.

Example:

<header role="banner"> </header>

The correlation between landmarks and the HTML5 semantic tags tend to be pretty straightforward, in some cases the role is the same as the tag.

Semantic tags and corresponding landmarks:

  • Main tag -> "main" role
  • Nav tag -> "navigaton" role
  • Header tag -> "banner" role
  • Aside tag -> "complementary" role
  • Section tag -> "region" role
  • Article tag -> "article" role
  • Form tag -> "form" role
  • Footer tag -> "contentinfo" role

There's also the "search" role that should be used in the search bar of a website.

Colors

We need to ensure that the colors for our website have enough contrast to be easily readable, because that's the purpose of text. If a text is not readable, our information won't be communicated to the users.

Color blindness:

Our website should also be accessible for the different types of color blindness that exists. Here is a list:

  • Protanomaly -> Red color weakness
  • Protanopia -> Red color blindness
  • Deuteranomaly -> Green color weakness
  • Deuteranopia -> Green color blindness
  • Tritanomaly -> Blue color weakness
  • Tritanopia -> Blue color blindness
  • Achromatopsia -> Monocromatic view

    • TIP: Test for Achromatopsia to check all colorblindness types and don't use only colors to convey meaning to the content.

Multimedia

Providing text alternatives to convey the meaning of our multimedia content is fundamental for users that browse using a screen reader or with some level of deafness.

Images

An alt attribute (alternative text) in our images should always be used .It should contain a small description of the image content or be left empty if the purpose purely decorative so screen readers skip it.

<img src="background.jpeg" alt="">
<img src="dogPlushW12.jpeg' alt="White dog plush of 12 centimeters">

Video

The first rule is to use a media player that's accessible and allows to be navigated using the keyboard, the second is to provide alternative text both for visual and hearing impaired users. Ideally, all of these should be provided:

  • Captions -> for users who can't hear the audio track, it should include the words spoken and contextual information and descriptions of what's happening.
  • Descriptions -> for users who can't see the video, it should include narrative descriptions of the content showcased.
  • Subtitles -> for users who don't understand the language used, should include translations of the visual and auditory content.

Text alternatives are written in .vtt format. Note that an informative default should be provided for users whose browser's is unable to play the video.

<video controls>
<source src="sample.mp4" type="video/mp4">
<source src="sample.ogv" type="video/ogv">
<track kind="captions" src="captions.vtt" srclang="en">
<track kind="descriptions" src="descriptions.vtt" srclang="en">
<track kind="subtitles" src="subtitles_en.vtt" srclang="en">
<track kind="subtitles" src="subtitles_es.vtt" srclang="es">
<p> Your browser doesn't support HTML5 video. </p>
</video>

Audio

Fairly similar to the video section, but only captions are needed since there's no visual content being showcased, only auditory.

<audio controls>
<source src="audio.ogg" type="audio/ogg">
<source src="audio.mp3" type="audio/mp3">
<track kind="captions" src="captions.vtt" srclang="en">
<p> Your browser does not support audio content. </p>
</audio>
